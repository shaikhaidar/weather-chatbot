+# вүЎЖ’Г®ГұвҲ©в••Г… weatherBOT Architecture & Training Fixes Report
+
+`weatherBOT` is a restricted, offline AI Weather Intelligence Platform engineered to run on isolated Edge hardware. It synthesizes Machine Learning metrics (Redacted), Spatial Delta predictions (PyTorch Graph Neural Networks), and localized Large Language Model inference (Ollama `llama3.1:8b`).
+
+---
+
+## вүЎЖ’Г…Г№вҲ©в••Г… Core Architecture Overview
+
+```mermaid
+graph TD
+    User([User UI / React Dashboard]) -->|HTTP REST| FastAPI[FastAPI Backend Engine]
+    FastAPI --> Router[Decision Engine Heuristic Router]
+    
+    subgraph Data & Background Training
+        FastAPI -->|Upload CSV| CSVProc[Dataset Service - Parsing & Alias Detection]
+        CSVProc -->|Save Meta| DB[(SQLite Database)]
+        CSVProc -->|Background Task| SelfLearning[Self-Learning ML Pipeline]
+        SelfLearning -->|Preprocess & Clean| CleanData[DataFrame Sanitization & Target Detection]
+        CleanData -->|Train Redacted| XGB[Redacted Regressor Model]
+        XGB -->|Promote if better RMSE| DB
+    end
+
+    subgraph Edge Inference Execution
+        Router -->|Determine Intent| ContextBuilder[Tri-Mode Context Builder]
+        DB -->|Fetch Active Model & Dataset Meta| ContextBuilder
+        ContextBuilder -->|Construct Edge System Prompt| Ollama[Local Ollama API - llama3.1:8b]
+        Ollama -->|Generate Natural Text + JSON Charts| FastAPI
+    end
+    
+    FastAPI -->|Response & Plotly Config| User
+```
+

+## вүЎЖ’Г¶Вә Comprehensive Bug Fixes & Technical Enhancements
+
+### 1. **Robust Redacted Training & Dynamic Hardware Auto-Detection**
+* **Location**: `backend/services/ml_service.py`
+* **Fix**:
+  * Added dynamic CUDA vs CPU hardware auto-detection via `torch.cuda.is_available()`. If GPU/CUDA is missing, training defaults safely to `"cpu"`.
+  * Sanitized `Inf` and target `NaN` values before model fitting (`df.replace([np.inf, -np.inf], np.nan)` and `.dropna(subset=[target_col])`) to eliminate Redacted DMatrix label crashes (`Label contains NaN, infinity or a value too large`).
+  * Replaced scikit-learn deprecated parameter `squared=False` in `mean_squared_error()` with `np.sqrt(mean_squared_error(...))`.
+
+### 2. **Intelligent Temperature Feature Matching**
+* **Location**: `backend/services/ml_service.py`
+* **Fix**: Instead of strictly requiring exact column header `"temperature"`, `MLService` checks `DatasetService.SENSOR_ALIASES['temperature']` (`temp`, `temp_max`, `temp_min`, `temp_c`, `temperature`). This prevents the system from misinterpreting arbitrary dataset columns (e.g. IDs or timestamps) as the weather target variable.
+
+### 3. **Database-Backed Training Status & Error Reporting**
+* **Location**: `backend/models.py`, `backend/schemas.py`, `backend/routers/datasets.py`
+* **Fix**:
+  * Added `status` (`PROCESSING`, `COMPLETED`, `FAILED`) and `error_message` fields to the SQLite `datasets` table schema.
+  * `run_self_learning()` background task updates dataset records directly. If training fails, the exact exception details are stored in the DB.
+  * Updated `RawDataset.tsx` in the frontend dashboard to display colored status badges (`COMPLETED` in green, `FAILED` in red with error tooltips, `PROCESSING` in yellow).
+
+### 4. **Enhanced Dataset Context for Edge LLM Inference**
+* **Location**: `backend/services/conversation_service.py`
+* **Fix**: Expanded LLM prompt context to include dataset filename, date span (`time_span`), total rows, and active sensor lists. When historical models are missing or training, explicit state notices are provided to Llama 3.1, eliminating hallucinations.
+
+### 5. **Tailwind CSS v4 & Datetime Warning Fixes**
+* **Location**: `frontend/src/index.css` & `backend/services/dataset_service.py`
+* **Fix**:
+  * Standardized `index.css` layer declarations for Tailwind CSS v4 styling.
+  * Updated `pd.to_datetime(..., format='mixed')` parsing in `DatasetService` to silence date format warnings.
+

+npm run dev
+```
